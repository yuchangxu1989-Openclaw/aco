🤖 助手: 好问题。确实有这个风险。

LLM 有锚定效应——你在 prompt 里显式说"建议使用 chart-craft-plus"，它很可能就直接用了，不再认真看其他 skill 的 description。如果 KIVO 推荐错了，反而比没推荐更差。

所以更好的做法可能是：**不做显式 skill 推荐**。KIVO 当前的注入方式已经是对的——注入的是知识上下文（规则、意图、方法论），不是 skill 名称。LLM 拿到更准确的意图理解后，自己从 skill 列表里选，选择质量自然提升。这是间接增强，没有锚定风险。

显式 skill 推荐只在一个场景下有价值：skill 数量多到 LLM 的 description 列表塞不下、必须预筛选的时候。目前我们几十个 skill，还没到那个规模。
