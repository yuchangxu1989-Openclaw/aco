## 发版任务路由决策（永久生效）

- 原生 subagent（dev-01/dev-02）的 exec 审批是 OpenClaw 内置行为，npm publish / git push --force 会弹审批，无用户可配置的全局开关

- 结论：发版任务优先派 ACP agent 或主会话直接执行，不派原生 subagent

- 主会话手动往 `subagent-task-board.json` 写入错误 schema（`taskId` 而非 `id`，缺 `createdAt` 等），导致 `reconcileStaleRunning` 抛异常，飞书卡片推送全挂 6 小时
- 规则固化：禁止直接写看板 JSON，必须通过 `local-subagent-board.js enqueue` 命令

## 编码 Agent 能力观察

- cc：架构理解强，但对大代码库读取+编码容易超时；拆小后效率高；域 W 第三次成功（前两次 free-code/opencode 失败）
- free-code：稳定可靠，承担了 KIVO 7 个域的编码；CD P1 修复一次成功
- hermes：能完成任务但容易遗漏关键步骤（如 npm run build）
- opencode：多次 0 token 实质失败，可靠性最差
- codex：completion 统计不可信但能写出结果
