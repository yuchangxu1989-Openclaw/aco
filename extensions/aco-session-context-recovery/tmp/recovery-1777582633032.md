核心思路很简单：DeepSeek 的创新点是让模型在推理过程中必须输出视觉坐标（"我说的是画面左上角这个人"），而不是含糊地说"那个人"。这个思路不需要等它的模型，用现有的 GPT-4V / Claude Vision / Gemini Vision 就能做。

具体做法：在 prompt 里强制要求模型输出结构化视觉标注 — 比如分析一张图时，必须给出每个被提及对象的 bounding box 坐标、位置描述、面积占比。这样模型的推理链就从"这张图有个人在走路"变成"[0.2, 0.3, 0.5, 0.8] 区域有一个人在向右走，占画面 24%"。

对我们立刻有用的场景：
- **Claw Design 视觉 QA**：生成设计稿后，让视觉模型用这种结构化方式检查构图、元素遮挡、留白比例
- **视频关键帧筛选**：抽帧后让模型标注每帧的主体位置和面积，自动判断哪些帧构图完整、主体清晰
- **智能裁切**：横版转竖版时，模型给出主体坐标，裁切就有依据了

实现成本很低，就是一套 prompt 协议 + JSON schema，不需要训练任何模型。

核心问题：Visual Primitives 的价值来自**模型训练层面** — DeepSeek 专门用 cold-start 数据训练模型在推理过程中使用坐标。这跟在 prompt 里加一句"请输出 bounding box"完全不是一回事。

GPT-4V / Claude Vision 被要求输出坐标时，精度很差。它们没有被训练成"边看边指"，只是在被逼着猜坐标。所以"抄方法论"在 prompt 层面能拿到的收益很有限，可能就是输出格式更结构化了，但空间精度不会有质的提升。

**能测，但测出来大概率会证明 prompt 层面不够用。**

测法：
1. 拿一个视频文件，用 video-frames 抽帧
2. 同一帧分别用两种 prompt 发给 Claude Vision：普通描述 vs 要求输出坐标+结构化标注
3. 对比两种输出的信息量和坐标准确性

要不要我现在就跑一个？拿一个视频文件实测一下，看看现有模型在结构化视觉标注上到底能做到什么程度。这样你能直接看到差距在哪，而不是我在这猜。
