### SEVO 拦截强化方案（用户讨论，待进 spec）
- 用户问"说明 SEVO 给你的注入还不够强，你觉得如何改？"
- 方案 1（推荐）：L2 确定性拦截 — spawn task 中出现 `projects/<tracked-project>/src/` 路径或构建命令 → 直接拦截，不过 LLM 判断
- 方案 2：L0 文件权限 — src/ 目录对普通 subagent 只读，只有 SEVO pipeline 阶段有写权限（重但最可靠）
- 方案 3（兜底）：L2 事后审计 — spawn 完成后检查 git diff 是否涉及 tracked project src/，无关联 pipeline 则回滚+通知

### 教训沉淀
- "一行改动"不能成为绕过 SEVO 的理由 — 这和 5/16 的 badcase 是同一类错误
- SEVO 拦截的 LLM 判断层有同类盲区：LLM 判断 LLM 是否该走流水线，天然会低估"简单改动"的流程价值
- 确定性拦截（路径匹配）比 LLM 判断更可靠，应该作为 SEVO 产品能力落地
