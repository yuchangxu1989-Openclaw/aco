# ACO — 多 Agent 协作的可控调度中枢

让 Agent 集群从「各自为战」升级为「协同可控」。

---

## 为什么需要 ACO

跑多个 Agent 协作时，你大概率遇到过这些问题：

任务派给了不该接的 Agent，失败了只能手动重派，主会话盯着看板等完成通知，推进链全靠人肉串联。调度规则写在文档里，模型一长上下文就忘了。Agent 卡死了没人知道，直到用户主动去查。开发 Agent 自己审自己的代码，质量门禁形同虚设。

ACO 把这些问题收拢到一套代码级强制执行的调度治理框架中——派发准入、角色约束、梯队路由、并发控制、超时熔断、失败重派、自动推进、IM 通知、全链路审计，全部可编程、可追溯、可观测。

---

## 核心能力

### 派发治理（dispatch-guard）

- **角色-任务匹配阻断** — 每个任务经 LLM 语义分类（spec/ac/code/audit/ux/readme/data-ops），只有对应角色的 Agent 才能接单。PM 写 spec，架构师写契约，编码 Agent 写代码，审计 Agent 做审查。派错了直接拦截，不靠模型自觉。
- **梯队路由** — 编码 Agent 分四级梯队（T1→T4），按任务复杂度和 Agent 空闲状态自动选择。同梯队失败后自动升级到下一梯队，不原样重派。
- **无偏见调度** — 禁止默认派某个 Agent。空闲优先、梯队顺序、配置驱动，消除调度偏见。
- **ACP 并发控制** — 最多 8 路 ACP 任务同时运行（可配置），超限自动排队，防止 OOM。
- **主会话保护** — 禁止向 main 派发任务，主会话只做秒级调度决策，不执行耗时操作。用户消息永远不会因为主会话忙碌而被丢弃。
- **开发审计分离** — 开发 Agent 不能自审。开发完成后自动触发独立审计 Agent 复查，代码级强制执行。
- **SEVO 流水线集成** — 自动识别 SEVO 流水线标签，按阶段映射任务类型，阻断绕过 SEVO 流程的非法派发。
- **超时下限强制** — 所有任务 timeout 不得低于 600 秒，防止因时间不足导致的虚假失败。
- **Agent 存在性校验** — 派发前实时读取 openclaw.json 验证 Agent 是否存在，不依赖缓存或历史印象。

### 运行时监控（run-watchdog）

- **超时熔断** — 任务运行超过阈值（默认 30 分钟）自动标记为 stale，触发告警和恢复流程。
- **卡死检测（idle-alert）** — 监控子 Agent 的 transcript 活动，5 分钟无新输出即告警，建议 kill + 拆分重派。
- **ACP 僵尸进程回收** — 终态 ACP 任务的残留进程自动 SIGTERM，5 秒后仍存活则 SIGKILL。
- **零 token 防御** — 任务报告成功但实际无输出（0 token、0 runtime），自动判定为实质失败，触发重派。
- **memory-core 熔断器** — memory_search 连续失败时自动熔断，防止级联超时拖垮整个调度链。
- **工具调用链追踪** — 每个子 Agent 的工具调用记录完整落盘，事后可审计每个任务做了什么。
- **自动推进通知** — 任务完成后向主会话注入推进提醒，单个 Agent 空闲即触发（不等所有任务归零）。
- **看板状态同步** — 子 Agent 生命周期（spawn → running → done/failed）实时同步到任务看板，飞书快照自动推送。

### 前置注入引擎（bootstrap hook）

- **角色上下文自动注入** — 每个 Agent 启动时自动加载对应的 SOUL.md（角色定义）、AGENTS.md（共享规则）、TOOLS.md（能力清单）等文件，确保子 Agent 拿到完整上下文。
- **终局门禁注入** — 所有 Agent（含 main）自动注入 TERMINAL_GATE.md，统一闭环标准。
- **路径遍历防护** — agentId 含 `..` 或路径分隔符时直接拒绝，防止注入攻击。

### 任务链编排

- **声明式推进链** — 定义多步骤任务串联（开发→审计→测试→视觉验证→发布），上一步完成自动触发下一步。
- **失败即时重派** — 收到失败 completion 时立即优化任务描述后重派，不等同批其他任务回收。同梯队失败禁止原样重派，必须拆分或升级。
- **长文档分段写作** — 预估超 300 行的文档任务自动拆段派发，避免单次超时失败。
- **并行任务文件隔离** — 多 Agent 并行时产出文件名自动包含 agentId，防止互相覆盖。

### 全链路可观测

- **决策审计日志** — 每个派发决策（allow/block/rewrite）带时间戳、规则 ID、原因写入 JSONL，可按任务/Agent/时间范围查询。
- **任务看板** — 实时状态聚合视图，支持持续刷新。
- **资源统计** — Agent 利用率、任务吞吐量、成功率、失败分布。
- **失败聚合分析** — 按 Agent、任务类型、时间段聚合失败模式，定位系统性问题。

### 通知推送

- **任务状态变化实时推送** — 完成、失败、卡死、超时等事件自动推送到配置的 IM 渠道。
- **Webhook 通道** — 可插拔的通知传输层，支持飞书、自定义 Webhook。
- **看板快照推送** — 关键状态变化时自动生成看板快照并推送。

### 健康与恢复

- **配置完整性检查** — 校验 Agent 池、数据目录、通知渠道的健康状态。
- **Gateway 恢复机制** — 检测到不可恢复的 stale 状态时，可选自动重启 Gateway（需显式开启）。
- **恢复锁** — 防止并发恢复操作互相冲突。

## 30 秒快速体验

```bash
# 安装
npm install -g @self-evolving-harness/aco

# 初始化配置（自动检测现有 dispatch-guard / run-watchdog 配置并迁移）
npx aco init

# 查看任务看板
aco task list

# 查看 Agent 资源池状态
aco pool

# 查看调度决策审计
aco audit --last 10

# 健康检查
aco health
```

## CLI 命令速查

| 命令 | 说明 |
|------|------|
| `aco task list` | 任务看板 — 实时状态聚合视图 |
| `aco dispatch` | 手动派发任务 — 经过完整治理校验 |
| `aco pool` | 资源池 — Agent 状态、梯队分布、忙闲情况 |
| `aco stats` | 统计 — Agent 利用率、任务吞吐量、成功率 |
| `aco failures` | 失败分析 — 按类型/Agent/时间聚合失败模式 |
| `aco audit` | 决策溯源 — 查询调度决策链和拦截记录 |
| `aco rule` | 规则管理 — 查看、添加、删除调度规则 |
| `aco chain` | 推进链 — 创建、查看、删除声明式任务链 |
| `aco config` | 配置管理 — 查看、校验、生成配置 |
| `aco notify` | 通知管理 — 通道注册、连通性测试 |
| `aco health` | 健康检查 — 配置完整性、数据目录、Agent 池 |
| `aco init` | 初始化 — 生成配置文件，迁移现有插件配置 |

所有命令支持 `--help` 查看详细用法，`--json` 输出机器可读格式。

## Agent 梯队体系

ACO 将编码 Agent 分为四级梯队，按任务复杂度和资源状态自动路由：

| 梯队 | Agent | 运行时 | 定位 |
|------|-------|--------|------|
| T1 | cc, free-code | ACP | 主力编码，高并发 |
| T2 | opencode, codex | ACP | 中等任务，备选 |
| T3 | hermes | ACP | 复杂任务，深度推理 |
| T4 | dev-01, dev-02 | Subagent | 兜底，轻量任务 |

非编码角色独立管理：架构师（sa-01/sa-02）、PM（pm-01/pm-02）、审计（audit-01/audit-02）、UX（ux-01）、调研（re）。

## 谁会用、什么场景

**多 Agent 平台运维者** — 管理 3 个以上 Agent 协作的系统，需要统一调度治理，避免错派、越权、资源浪费。

**AI 工程团队** — 构建 Agent 协作流水线，需要声明式任务链编排和故障自愈，不想每次失败都人工介入。

**单 Agent 用户** — 即使只有一个 Agent，也能获得超时保护、失败重试、任务看板和状态通知。

典型场景：
- 研发流水线中多 Agent 分工协作（开发→审计→测试→发布）
- Agent 集群的并发控制和负载均衡
- 任务失败后的自动梯队升级和重试
- 调度决策的事后审计和问题排查
- 主会话保护：确保用户消息不因调度阻塞而丢失
- 僵尸进程治理：ACP 残留进程自动回收

## 编程接口

ACO 同时提供 Node.js SDK，可嵌入到现有系统中：

```typescript
import { createScheduler } from '@self-evolving-harness/aco';

const scheduler = createScheduler({ configPath: './aco.config.json' });
await scheduler.dispatch({ label: 'code-review', prompt: '...' });
```

插件模式（作为 Gateway 插件加载）：

```typescript
import { createPlugin } from '@self-evolving-harness/aco/plugin';
```

## 渐进式配置

ACO 按 Agent 数量自动解锁功能层级：

- **单 Agent**：超时保护、失败重试、任务看板、IM 通知
- **2-5 Agent**：角色约束、梯队路由、开发审计分离
- **5+ Agent**：并发控制、负载均衡、全链路审计、推进链编排

零配置即可启动，`aco init` 自动检测环境生成最优配置。
